# Legacy Superpowers Design History

This directory preserves design material created under the historical
Superpowers workflow.

Keep these files for links, rationale, and chronology, but do not treat this
tree as the required orchestration layer for new AOS work. New provider-neutral
AOS plans, specs, notes, and supporting artifacts should start under
[`../design/`](../design/) unless a task is deliberately continuing one of
these legacy threads.

When stable decisions from a legacy plan or spec become current behavior,
promote them into the appropriate durable home:

- `ARCHITECTURE.md` for system architecture
- `docs/api/` or `shared/schemas/` for cross-tool contracts
- `docs/recipes/` for repeatable operating procedures
- `wiki-seed/` for source-controlled runtime wiki seed knowledge
- GitHub issues and PRs for unresolved work and execution history
