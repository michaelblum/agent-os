## Required Behavior

### Workbench producer record

- Delete or fold `packages/toolkit/workbench/html-workbench-expression.js`
  `semanticTarget()` into the canonical normalizer path. The generated
  semantic target metadata must use the canonical `agent_ui_target` producer
  shape from Slice 2.
- Generated workbench semantic target records must expose a single top-level
  identity key: `ref`.
- Do not emit top-level `target_id`, `aos_ref`, `data_aos_ref`,
  `accessible_label`, `semantic_target_id`, `subject_id`, or `do_target` from
  generated workbench producer records.
- Preserve workbench-specific data losslessly under `extension` and
  `provenance`:
  source path, source line start/end, kind, annotation/reveal eligibility,
  selector/reveal hint, and any stable DOM slug needed to render
  `data-semantic-target-id`.
- Keep selectors and DOM ids as reveal/provenance hints only. Selectors must not
  become input identity.

### Generated HTML and fixtures

- Rendered HTML should continue to stamp `data-aos-ref`,
  `data-aos-surface`, `data-semantic-target-id`, source line attributes, and
  target kind attributes so existing reveal/annotation mechanics remain
  inspectable.
- The value of `data-aos-ref` must come from canonical `ref`.
- If `data-semantic-target-id` needs the old slug, derive it from canonical
  target provenance or extension. Do not reintroduce top-level `target_id`.
- Update checked-in generated fixtures under
  `docs/design/fixtures/aos-html-workbench-expression-v0/` and the Employer
  Brand human-alignment expression fixtures when deterministic tests require
  it.

### Source maps, sidecars, and resume payloads

- Migrate source maps, mermaid block metadata, checkpoint/resume sidecars, and
  decision/annotation records generated by this adapter to use canonical `ref`
  for target joins.
- If a field must remain named differently because it is an external or
  historical sidecar contract, stop and report that boundary. Do not silently
  preserve `target_id` as a compatibility alias.
- Preserve source path and line-range lookup behavior exactly.

### HTML workbench component consumer

- Update `packages/toolkit/components/html-workbench-expression/index.js` to
  resolve/reveal canonical workbench target records by `ref`.
- It may still use `selector` and rendered `data-aos-ref` / `data-semantic-target-id`
  attributes as reveal hints. It should not require `target_id`,
  `data_aos_ref`, or `aos_ref` on new producer records.
- Keep offscreen reveal, late Surface Inspector attach replay, and explicit
  blockers working.

### Projection and Surface Inspector consumers

- Update `buildSemanticTargetProjectionAdapterResult(...)`,
  `buildRevealPayloadForSurfaceInspectorPin(...)`, and the Surface Inspector
  eval selector fallback only as needed to consume canonical `ref` workbench
  targets.
- Do not collapse projection records into producer records.
- Do not remove deliberate consumer/projection vocabulary such as `subject_id`,
  `subject_path`, `root_id`, `current_render_status`, and
  `display_space_rect`.
- Do not collapse browser adapters or settle the review Section 11 open
  questions.

### Conformance fixture pack

- Update `docs/design/fixtures/agent-ui-target-conformance-v0/` and
  `tests/toolkit/agent-ui-target-conformance.test.mjs` so the workbench source
  target is no longer the remaining producer drift.
- Keep the projection cardinality gate: at least two projection records for the
  same canonical `ref`, keyed by `(adapter_id, ref)`.
- After this slice, the conformance mapping table should not list workbench
  `target_id`, `aos_ref`, or `data_aos_ref` as a current producer identity
  spelling.

  Playwright dialect.
- Do not run live pointer or supervised Operator checks for this deterministic
  migration.

## Stop Conditions

Stop with a clear report instead of continuing if:

- source-line/checklist/mermaid/document workbench targets cannot be represented
  as canonical `agent_ui_target` records without top-level `target_id`;
- resume sidecars or source-map consumers require an external compatibility
  boundary that Foreman must decide;
- Surface Inspector reveal requires selector-as-identity instead of canonical
  `ref` plus selector provenance;
- the migration would require changing live DOM attribute names.
