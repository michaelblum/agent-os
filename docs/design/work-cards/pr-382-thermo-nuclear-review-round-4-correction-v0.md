## Review Finding To Correct

### Supplied keyframe asset refs can be masked before validation

`src/daemon/surface-inspector-bundle.swift` validates the final merged
`context_keyframe`, but the supplied keyframe's `asset_refs` are merged with
daemon-generated bundle asset refs first:

```swift
var mergedAssetRefs = keyframe["asset_refs"] as? [String: Any] ?? [:]
for (key, value) in assetRefs {
    mergedAssetRefs[key] = value
}
keyframe["asset_refs"] = mergedAssetRefs
try rejectEmbeddedContextAssetRefs(keyframe, path: "context_keyframe")
```

That means an invalid supplied keyframe ref whose key is also generated by the
daemon, such as `capture_image`, `capture_json`, `display_geometry_json`,
`canvas_list_json`, `inspector_state_json`, or
`surface_inspector_annotation_snapshot`, is overwritten before validation and
does not fail closed.

Example payload that should be rejected but is currently masked:

```js
{
  context_session: {
    schema: "aos_context_session",
    version: "0.1.0",
    id: "context-session:valid",
    artifacts: [],
    keyframes: []
  },
  context_keyframe: {
    schema: "aos_context_keyframe",
    version: "0.1.0",
    id: "keyframe:invalid",
    captured_at: "2026-05-28T12:00:00.000Z",
    trigger: "invalid",
    artifact_ids: [],
    asset_refs: {
      capture_image: " Data:text/plain;base64,SGk="
    }
  }
}
```

This violates the round-3 acceptance contract: supplied context asset refs must
match the canonical JS/schema external-asset contract before the daemon writes
or normalizes them. Do not treat overwriting a bad supplied ref as successful
sanitization.

Required correction:

- Validate supplied `context_keyframe` asset refs before merging daemon-generated
  asset refs.
- Keep the post-merge validation as a defense against future generated-ref
  regressions.
- Preserve structured failure:
  `CONTEXT_PAYLOAD_INVALID_ASSET_REF` with phase
  `context_payload_validation`.
- Add coverage where the invalid supplied `context_keyframe.asset_refs` key
  collides with a generated daemon asset key. The test must fail without the
  code fix.
- Include at least one leading-whitespace `data:` value or one `blob:` object
  URI in the colliding key case.
- Prefer extending the existing live bundle-path and clipboard-payload scripts
  if that stays bounded. If only one live path can be covered cleanly, add a
  deterministic daemon/toolkit assertion for the other path and explain why.
