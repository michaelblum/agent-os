# Report Folder Structure

Every report is a self-contained folder. All paths inside `index.html` and `report-data.json` are **relative to the folder root**, so the folder can be moved, zipped, or uploaded to any static host without modification.

---

## Canonical Layout

```
{client-slug}-audit-report-{YYYY-MM-DD}/
│
├── index.html                         ← The interactive SPA (generated by this plugin)
├── report-data.json                   ← The structured data file
│
└── assets/
    └── images/
        │
        ├── _system/                   ← Shared system assets
        │   ├── st-logo.png            ← Symphony Talent wordmark / logo
        │   ├── hero-bg.jpg            ← Background image for Overview section
        │   ├── content-bg.jpg         ← Background image for Summary + Deep Dive sections
        │   └── kilos-watermark.png    ← Optional decorative KILOS logo (used in hero overlay)
        │
        ├── {client-slug}/             ← One folder per company
        │   ├── logo.png               ← Company logo (ideally transparent PNG, min 300px wide)
        │   ├── {domain-slug}/         ← One subfolder per evidence domain
        │   │   ├── img-001.png
        │   │   ├── img-002.png
        │   │   └── ...
        │   └── linkedin-com/
        │       ├── img-001.png
        │       └── ...
        │
        ├── {competitor-a-slug}/
        │   ├── logo.png
        │   └── {domain-slug}/
        │       └── img-001.png
        │
        └── {competitor-b-slug}/
            └── ...
```

## Naming Conventions

### Report folder name
```
{client-slug}-audit-report-{YYYY-MM-DD}
```
Example: `acme-corp-audit-report-2026-03-19`

### Company slugs
Lowercase, spaces → hyphens, special chars removed.

| Company name | Slug |
|---|---|
| Acme Corp | `acme-corp` |
| GlaxoSmithKline Pharmaceuticals (GSK) | `glaxosmithkline-pharmaceuticals-gsk` |
| Johnson & Johnson | `johnson-johnson` |
| AstraZeneca | `astrazeneca` |

### Domain slugs (for evidence subfolders)
Take the primary domain, lowercase, dots → hyphens.

| Domain | Slug |
|---|---|
| `careers.acme.com` | `careers-acme-com` |
| `glassdoor.com` | `glassdoor-com` |
| `linkedin.com` | `linkedin-com` |
| `indeed.com` | `indeed-com` |
| `x.com` | `x-com` |

### Image filenames
Use sequential numbering: `img-001.png`, `img-002.png`, etc.
For logos: always `logo.png`.
For system assets: descriptive names (`st-logo.png`, `hero-bg.jpg`, etc.).

## Image Specifications

| Asset | Format | Recommended size | Notes |
|-------|--------|-----------------|-------|
| Company logo | PNG (transparent) | 400×200px min | Used at h-16 (64px) in deep dives; h-7 (28px) in sub-nav; max-h-12 in competition grid |
| System logo (ST) | PNG (transparent) | 300×80px min | Used at h-8 (32px) in header, h-20 (80px) in hero |
| Background images | JPG | 1920×1080px | Displayed behind dark overlay; quality over resolution |
| Evidence screenshots | PNG or JPG | Any | Displayed at h-80 (320px) in carousel; clicked to open in lightbox |
| KILOS watermark | PNG (transparent) | 400×400px | Displayed at low opacity in hero overlay |

## `localPath` Format in `report-data.json`

All `localPath` values are written as paths **relative to the report folder root** (where `index.html` lives):

```json
{
  "localPath": "assets/images/acme-corp/careers-acme-com/img-001.png"
}
```

Do **not** use leading `/` or `./` — just the bare relative path.

## Online Deployment

To deploy a report online, upload the entire folder to any static host:

### Netlify / Vercel
Drop the folder into the web UI or run:
```bash
netlify deploy --dir=acme-corp-audit-report-2026-03-19
```

### AWS S3
```bash
aws s3 sync acme-corp-audit-report-2026-03-19/ s3://your-bucket/reports/acme-corp-2026/ --acl public-read
```

### GitHub Pages
Commit the folder to a repo's `gh-pages` branch or `/docs` directory.

### CDN prefix (optional)
If you want to serve images from a CDN while keeping `index.html` on a different origin, set `reportData.reportMeta.imageBase` to the CDN URL:
```json
"imageBase": "https://cdn.example.com/reports/acme-corp-2026"
```
The `lUrl()` function in the HTML template will prepend this to all image paths.

## Portability Checklist

Before distributing or archiving a report, verify:

- [ ] `index.html` and `report-data.json` are in the root of the report folder
- [ ] All `localPath` values in `report-data.json` resolve to actual files
- [ ] `assets/images/_system/` contains `st-logo.png`, `hero-bg.jpg`, `content-bg.jpg`
- [ ] Every company that appears in the data has a `logo.png`
- [ ] The folder opens correctly locally: `open index.html` or `python3 -m http.server`
- [ ] `imageBase` is set correctly for the deployment environment (empty for local)
